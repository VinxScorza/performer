PER|FORMER

test drive

forked and maintained by Vinx Scorza

Description

This simulator is used for development and testing of the PER|FORMER sequencer. It allows shorter iteration times during development and makes it possible to work on the firmware without having direct access to the hardware module.

Note that this simulator is not intended to replace the hardware unit, and timing accuracy is not comparable to the real module, especially when running in the browser.

This page belongs to the Vinx Scorza fork site and keeps the simulator reference close at hand while the firmware evolves.

The current browser build reflects the active Vinx local line, including the newer generator workflow, the refined Random / Acid LCD previews, and the current simulator demo/audio setup.

Historical simulator reference: Mebitek Test Drive and Westlicht Test Drive

We are actively trying to bring the browser Test Drive back to life for the Vinx Scorza fork.

Upstream browser builds currently fail at runtime for different reasons, so this page is being prepared for a local Vinx build that favors stability first: memory growth enabled, extra runtime checks during bring-up, and browser audio reintroduced cautiously behind explicit user controls.

Phase 2 Status

The local Vinx browser build is now embedded below with a cleaner simulator shell focused on the module itself.

The generator pages in this build use the newer 64-step preview layouts, framed bank highlighting, slimmer playhead, and refined Random / Acid rendering introduced in the recent Vinx line, including the newer layer-aware Random preview behavior on Note tracks.

Audio is now attempted automatically on startup, while still falling back gracefully to the first user interaction if the browser blocks autoplay.

This browser Test Drive should be treated as a convenient reference build, not as a perfect representation of the hardware module or the desktop simulator. Timing, audio behavior, browser input handling, and some UI details may still differ from the real firmware experience on hardware or in the native simulator.

If the embed does not start correctly in your browser, open the standalone build: local simulator shell.

Sound Engine

In the browser simulator, a small sound engine is attached to the sequencer outputs so the generated gates and CV can be heard immediately.

The current local build uses the refreshed Vinx demo state and the current Vinx Test Drive sound map:

  • TRK1: Kick
  • TRK2: Snare
  • TRK3: Clap
  • TRK4: Loop
  • TRK5: Closed hi-hat
  • TRK6: Open hi-hat
  • TRK7: Acid synth
  • TRK8: Curve lane controlling Run Mode

Notes

The current browser path is being hardened around cleaner embedding, automatic audio bring-up where the browser allows it, a refreshed demo state, and more reliable re-entry into a known simulator setup.

Browser behavior can still vary between engines. If keyboard control feels unresponsive after clicking elsewhere, use the simulator toolbar to focus the canvas again.

Even when the browser build is working correctly, always consider the hardware module and the native simulator as the final reference for timing, responsiveness, and exact behavior.

Vinx Scorza fork based on the Mebitek fork of the Westlicht Performer firmware.
